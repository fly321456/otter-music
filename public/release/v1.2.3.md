# OtterMusic Release Note - v1.2.3

本次更新专注于 **Android 原生体验优化**、**播放稳定性**以及**存储管理的规范化**。我们重构了本地音乐插件与下载系统，并提升了应用在弱网或文件丢失场景下的自愈能力。

---

## 🚀 新特性与优化

### 1. Android 原生能力重构 (Native Bridge)

- **精简 LocalMusicPlugin**: 深度重构 Android 原生插件，移除样板代码，优化 MediaStore 扫描逻辑，大幅提升本地歌曲索引速度。
- **Gradle 构建优化**: 引入阿里云 Maven 镜像加速国内环境下的编译速度。
- **自动化发布脚本**: 新增 `sign-apk.ps1` 自动化签名工作流，支持对齐 (zipalign) 与 V2/V3 签名校验。

### 2. 播放稳定性与同步

- **失效文件自愈**: 当播放已下载歌曲但检测到本地文件被物理删除时，应用将自动清除错误记录并尝试从网络流无缝重试播放。
- **状态同步优化**: 重构 `useAudioEventHandlers`，通过直接读取 Store 状态解决异步重试过程中的播放按钮/加载状态不同步问题。
- **播放进度恢复**: 将 `duration` 纳入持久化白名单，解决重启应用后底部播放栏进度条初始化显示为 0 的问题。

### 3. 存储与下载管理 (Storage Manager)

- **路径规范化**: 引入统一的 `storage-manager` 配置中心，规范化本地音乐、数据记录和缓存的存储路径。
- **下载记录持久化**: 下载记录由本地磁盘 JSON 文件驱动，不再完全依赖内存持久化，提升了记录的可靠性。
- **元数据解析优化**: 优化本地文件歌手名解析，支持多分隔符处理，并优先保留用户自定义的“歌手 - 标题”格式。

---

## 🛠 修复与改进

- **下载权限修复**: 针对 Android 10+ 引入 `requestLegacyExternalStorage`，修复低版本设备下载目录创建失败的问题。
- **歌单深度去重**: 提供更加智能的歌单去重逻辑，支持合并喜欢状态并优先保留已下载版本。
- **UI 交互**:
- `addToFavorites` 逻辑解耦，移除 Store 内部 Toast，由 UI 层统一控制反馈。
- 全局 Toast 默认持续时间延长至 3000ms。
- 修复应用静默更新检查逻辑，避免重复弹窗。
