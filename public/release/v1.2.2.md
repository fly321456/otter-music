# v1.2.2 Release Notes

**主要更新**
本次更新修复了更新后仍提示的bug
重构了应用更新检查机制，将版本比对逻辑迁移至前端，并显著优化了静默检查体验与更新对话框的视觉表现。

## 优化 (Improvements)

- **更新检查逻辑**: 将版本号比对从服务端迁移至前端处理，移除服务端 `hasUpdate` 依赖，增强了对离线或缓存场景的兼容性 (cd173f4)
- **版本比对算法**: 新增版本号标准化工具函数，支持更精准的语义化版本号（SemVer）比对 (cd173f4)
- **静默检查优化**: 优化了静默检查的触发逻辑，有效避免在同一会话中发起重复的更新请求 (cd173f4)
- **对话框体验**: 增强了更新对话框的独立性，改进了版本信息展示逻辑，提供更清晰的更新指引 (cd173f4)

## 代码重构 (Refactoring)

- **状态管理**: 重构 `app-store.ts`，移除了冗余的 `hasNewVersion` 状态，统一由前端计算结果驱动 UI (cd173f4)
- **组件解耦**: 提取并优化 `UpdateDialog.tsx`，使其更具通用性并便于后续维护 (cd173f4)
