## v1.2.4 更新日志

### 🚀 新特性

- **新增 Bilibili 音源**：扩展了音乐搜索与播放的来源渠道。
- **存储引擎升级**：底层存储由 `localStorage` 迁移至 **IndexedDB**（基于 `idb-keyval`），解决容量限制问题并避免阻塞主线程，支持旧数据自动迁移。
- **引入自动化测试**：集成 Vitest 测试框架及相关配置，提升核心逻辑的稳定性。

### ⚡️ 优化

- **搜索算法精进**：
- **精确匹配**：搜索专辑时新增歌手维度匹配，大幅提升同名专辑的搜索准确性。
- **排序优化**：引入**威尔逊区间算法**动态计算音源质量，结合对数热度权重，使搜索结果排序更科学。
- **性能提升**：优化了搜索结果的去重与聚类算法，并分离了搜索触发与输入框聚焦逻辑。

### 🛠️ 修复与重构

- **类型安全增强**：大规模将 `any` 替换为 `unknown` 并使用类型守卫，完善了 `music-api` 等模块的接口定义。
- **交互修复**：修复了本地 (local) 音轨在跳转歌手或专辑时错误设置搜索源的问题。
- **文档更新**：更新了 README 目录结构描述及技术实施计划文档。
